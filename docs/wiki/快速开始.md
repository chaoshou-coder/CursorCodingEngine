# 快速开始

## 一、前置要求

- **Python 3.10+**：codingengine 为 Python 包
- **Cursor**（可选）：用于 Skill 驱动；CLI 可独立使用
- **Git**：用于 clone、原子提交与回滚

## 二、安装与验证

### 2.1 克隆并安装

```bash
git clone https://github.com/chaoshou-coder/CursorCodingEngine.git
cd CursorCodingEngine

# 创建虚拟环境，建议命名 CCEfor<项目名>
python -m venv CCEforCursorCodingEngine

# 激活（Windows PowerShell）
.\CCEforCursorCodingEngine\Scripts\Activate.ps1
# 激活（Linux/macOS）
# source CCEforCursorCodingEngine/bin/activate

# 安装
pip install -e .
```

### 2.2 验证安装

```bash
codingengine --help
# 应看到 init、emit、run、status、tail、list、stats 等子命令

pip show codingengine
# 确认 Location 指向 CursorCodingEngine
```

若 `codingengine` 命令不可用，使用 `python -m codingengine.cli` 或短别名 `ce`。

## 三、第一次运行（最简流程）

### 3.1 初始化

```bash
codingengine init "My first task"
```

从输出中取得 `run_id`（如 `20250305_120000_abcd1234`），后续命令均需带上。

### 3.2 执行四个阶段

| 阶段 | 命令 | 说明 |
|------|------|------|
| plan | `codingengine emit stage.started --stage plan --run-id <run_id>` | 开始规划 |
| plan | 分析需求，编写 plan.json 保存到 `~/.codingengine/runs/<run_id>/artifacts/plan.json` | 产出计划 |
| plan | `codingengine emit stage.completed --stage plan --run-id <run_id>` | 完成规划 |
| develop | `codingengine emit stage.started --stage develop --run-id <run_id>` | 开始开发 |
| develop | 按 plan.json 执行任务：修改 → lint → code-simplifier → `git commit -m "[codingengine] <task_id>"` | 逐任务开发 |
| develop | `codingengine emit stage.completed --stage develop --run-id <run_id>` | 完成开发 |
| verify | `codingengine emit stage.started --stage verify --run-id <run_id>` | 开始验证 |
| verify | 运行 `ruff check .`、`pytest`，失败则诊断回滚重试 | 全量验证 |
| verify | `codingengine emit stage.completed --stage verify --run-id <run_id>` | 完成验证 |
| 完成 | `codingengine emit run.completed --run-id <run_id>` | 结束 run |

### 3.3 预期输出

- **数据目录**：`~/.codingengine/runs/<run_id>/`
- **产物**：`artifacts/plan.json`、`artifacts/code_changes.json`、`artifacts/verify_result.json`
- **事件流**：`events.jsonl`
- **项目目录**：零新增文件，不污染项目

## 五、在任意项目中使用

在任意项目目录执行 `codingengine init "任务"`，项目内不会创建额外数据目录。所有数据存储在 `~/.codingengine/`，可通过 `CODINGENGINE_DATA` 覆盖。

**典型流程**：

1. 进入目标项目根目录
2. 激活该项目的 venv（建议 `CCEfor<项目名>`）
3. 安装 codingengine：`pip install -e /path/to/CursorCodingEngine`
4. 执行 `codingengine init "简短英文描述"`
5. 按主工作流 Skill 或手动 emit 执行 plan → develop → verify

## 六、与 Cursor Skill 配合

若在 Cursor 中使用，本项目已包含 `.cursor/skills/codingengine/` 和 `code-simplifier/`。在 Cursor 中打开本项目即可。

若要在**其它项目**中使用，需将 `CursorCodingEngine/.cursor/skills/` 复制到目标项目的 `.cursor/skills/`。主工作流 Skill 会引导 Agent 按阶段调用 `codingengine emit`，并融入 OMO Just Do、Boulder 续航、ECC 质量门禁。

详见 [Skill 系统](Skill系统.md)。
